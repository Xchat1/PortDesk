# PRD: PortDeck — macOS 本地服务控制台

PortDeck 是一款面向 macOS 开发者的本地开发服务控制台，帮助开发者一眼看清端口、进程、项目来源，并优雅地关闭无用的本地开发服务。

---

## 1. 产品概述

### 1.1 背景与痛点
开发者在 macOS 上进行本地开发时，经常会启动各种本地 Web 服务器、数据库、代理服务和 AI 网关（如 Node.js, Python, PostgreSQL, Go, Docker, Ollama 等）。由于这些服务通常在后台运行或运行于各终端窗口中，经常导致以下问题：
* **端口占用冲突**：想启动新服务时提示端口已被占用，不得不去终端执行 `lsof -i :3000` 并强制 Kill。
* **资源静默蚕食**：下班或关闭 IDE 后，后台残留的开发服务仍在默默消耗 CPU、内存和电池，导致 Mac 莫名发热耗电。
* **来源难以追踪**：终端里只能看到 `node` 进程占用了端口，不知道它是哪个项目启动的，也不敢贸然杀掉。

### 1.2 产品定位
**面向开发者的 macOS 本地服务控制台：一眼看清端口、进程、项目来源，并安全关闭无用开发服务。**
垃圾清理（如回收站容量统计）作为辅助的开发者日常清理工具，不喧宾夺主。

### 1.3 核心卖点 (USP)
* **项目来源感知 (Project-aware Port Detection)**：不只展示端口和 PID，更通过多因子分析找出该进程所对应的工作目录（如 `~/Projects/my-next-app`）、启动源（如 Cursor、VS Code、Terminal）以及项目类型。
* **优雅关闭 (Gentle Stop)**：采用“先礼后兵”的进程关闭逻辑，先发送 `SIGTERM` 允许程序保存状态，若超时未关闭再提示 `SIGKILL`。
* **超轻量与低能耗**：采用纯原生 **SwiftUI** 构建，没有 Electron/Tauri 的重度运行时，内存占用低，且不注册常驻后台守护进程。

## 2. 用户画像与核心场景

### 2.1 目标用户
* 前端/后端/Full-stack 开发者、AI 开发者。
* 频繁使用 Terminal / 各种 IDE 运行本地服务的技术人员。
* 对 Mac 续航、资源占用有较高要求的敏捷开发者。

### 2.2 核心场景
1. **下班前的“服务大扫除”**：关闭 IDE 后，打开 PortDeck，一键扫视当前所有残留的 Node/Python/Ollama 本地服务，将它们一键优雅关闭，让 Mac 零负担进入睡眠。
2. **解决端口冲突**：运行 `npm run dev` 报错端口 `8080` 被占用，打开 PortDeck，看到是上周的一个 Python 测试项目在后台没退干净，直接点击“优雅关闭”。
3. **空间回收**：开发完毕后，快速看看 Trash 里积攒的 DerivedData 或临时垃圾有多少，直接一键唤起系统清理。

## 4. 功能详细设计

### 4.1 Dashboard (仪表盘看板)
* **顶部摘要**：
  * Active Services: 展示当前监听的 TCP 端口数量。
  * Exposed Externally: 监听在 `0.0.0.0` 或 `*` 等对外暴露的服务数量（标记为黄色/橙色注意）。
  * Trash Occupied: 显示当前废纸篓的大小（例如：`1.2 GB`）。
* **卡片式布局**：
  * **本地服务快照**：高亮展示最近活跃的开发服务（如 localhost:3000）。
  * **能耗警示卡片**：如果检测到某些开发进程（如 node/python）占用了非正常的 CPU/内存，提示其可能正在默默耗电。
  * **轻量清理卡片**：展示 `~/.Trash` 大小，提供一个 "Open Trash" 按钮。

### 4.2 Ports & Services (服务控制台)
提供两种视图：**普通卡片视图 (Card View)** 与 **高级表格视图 (Advanced Table)**。

#### 普通卡片视图 (Card View - 推荐)
每张卡片代表一个正在监听的本地服务，包含：
* **框架图标/类型**：根据多因子规则识别出的服务类型（如 React 图标，Python 图标，NodeJS 图标）。
* **连接信息**：如 `localhost:3000` 或 `0.0.0.0:8000` (加警告标识)。
* **项目路径**：如 `~/Workspace/my-next-app`（点击可在 Finder 中打开或复制）。
* **启动源**：如 `Parent: Cursor (Terminal)`。
* **启动命令**：如 `npm run dev`。
* **操作键**：最右侧提供明显的 "Stop" 按钮。

#### 高级表格视图 (Advanced Table)
以传统的系统管理器网格展示：
* 列名：`PID` | `Service / Process` | `Address` | `Port` | `User` | `Parent` | `Command`
* 支持点击列头进行排序，以及全局关键字搜索（支持按端口号、进程名、工作目录搜索）。

### 4.3 Process Detail Drawer (进程详情抽屉)
点击任意服务卡片或表格行，右侧滑出抽屉展示更深维度的进程信息：
* **基本信息**：PID, 父进程 ID (PPID)，运行用户。
* **工作目录 (Cwd)**：进程运行时的实际绝对路径（支持一键在终端中打开该路径 `cd` 过去）。
* **完整启动命令**：换行折行展示完整的命令行参数。
* **进程链拓扑**：展示父子进程关系（例如：`launchd` -> `Cursor` -> `zsh` -> `node`）。
* **风险评估**：根据暴露范围、签名状态提供人性化解释（如：“此服务仅限本机访问，属于安全的本地开发环境”）。

### 4.4 Gentle Stop (优雅关闭机制)
点击关闭服务时，执行以下策略：
1. **二次确认**：弹窗提示“确定要关闭此服务吗？”并展示其关联的端口和项目名。
2. **发送 SIGTERM**：调用系统 `kill -15 <PID>` 信号，状态显示为 "Stopping..." 并开始 3 秒倒计时。
3. **检测与降级**：
   * 若 3 秒内进程自动退出了，则提示成功并刷新列表。
   * 若 3 秒后进程仍然存在，界面转换为 "Force Stop" (强杀) 状态，按钮变为红色。用户点击后，发送 `kill -9 <PID>` (SIGKILL) 强杀进程。
4. **日志记录**：将所有被用户关闭的进程操作（时间、PID、命令、项目路径、退出状态）写入本地操作日志，便于开发者回溯。

### 4.5 Trash / Lightweight Cleanup (轻量清理)
* **容量统计**：统计用户家目录下 `~/.Trash` 的总文件数量和字节大小，转换为人性化单位（MB/GB）。
* **打开回收站**：提供 `Open Trash` 按钮，通过系统命令调用 Finder 打开废纸篓。
* **清空操作**：
  * 点击 `Empty Trash` 时，弹窗请求用户确认。
  * 用户确认后，通过 AppleScript 调度系统 Finder 执行清空操作，或者调用系统安全提示，保证垃圾清理行为安全透明，不破坏系统权限。

## 5. 风险识别与服务类型推断规则

### 5.1 服务类型推断 (Multi-factor Framework Identification)
为了准确翻译“这个服务是做什么的”，采用多因子识别算法，而不仅仅依赖端口：

| 识别类型 (Framework) | 进程名 (Process) | 命令行关键字 (Command Arguments) | 关键特征文件 (Workspace Features) |
| :--- | :--- | :--- | :--- |
| **Next.js** | `node` | 包含 `next-server` 或 `next dev` | 工作目录下有 `next.config.js` |
| **Vite / Vue / React** | `node` | 包含 `vite` 或 `react-scripts` | 工作目录下有 `vite.config.ts` |
| **FastAPI** | `python` / `python3` | 包含 `uvicorn` 或 `fastapi` | 工作目录下有 `main.py` 或依赖含有 `fastapi` |
| **Django** | `python` / `python3` | 包含 `manage.py runserver` | 工作目录下有 `manage.py` |
| **PostgreSQL** | `postgres` | 包含 `postgres` | - |
| **Ollama** | `ollama` | 包含 `ollama` 或 `ollama serve` | - |
| **Docker** | `com.docker.backend` / `vpnkit` | - | - |
| **Java Spring**| `java` | 包含 `spring-boot` 或 `-jar` | - |

> **注**：端口仅用作展示和排序，推断类型时综合以上多因子判定；若都未匹配，则默认显示为“通用 <进程名> 服务”。

### 5.2 风险与暴露警告规则
* **低风险 (Local Only)**：绑定在 `127.0.0.1` 或 `[::1]` (IPv6)，属于本机闭环访问。
* **警告 (Exposed)**：绑定在 `0.0.0.0` 或 `*`。若局域网内有其他用户，可能通过 Wi-Fi 访问到此开发端口。在界面以黄色三角标志警告。

## 6. 权限与安全边界

为保证 PortDeck 是一个安全的开发助手，而非越权工具，必须坚守以下安全红线：
1. **无 Root 提权**：默认只扫描和展示当前登录用户（如 `admin` 或当前 developer 账号）名下的进程。不请求 `sudo`。
2. **系统进程免扰**：对于可执行文件路径位于 `/System/`、`/usr/sbin/`、`/bin/` 的进程，默认在 UI 中打上“系统”标签，并**禁用/屏蔽 Stop 操作**。
3. **Apple 签名保护**：对拥有 Apple 官方签名的进程，不建议用户执行任何关闭操作，隐藏强制杀死选项，避免误杀系统关键服务。
4. **操作安全确认**：清空废纸篓、SIGKILL 等破坏性操作必须通过标准 macOS 二次确认弹窗进行阻断，防止误触。

## 7. 技术方案设计

### 7.1 技术栈
* **架构**：原生 macOS 桌面应用程序 (macOS 14.0+)
* **语言**：Swift 5.9+ / 兼容 Swift 6 严格并发
* **UI 框架**：SwiftUI + 部分 AppKit 桥接 (用于菜单栏、Finder 交互等)
* **编译平台**：优先适配 Apple Silicon (arm64)，兼顾 Universal Binary (x86_64 + arm64)。

### 7.2 数据采集实现
1. **本地监听端口扫描**：
   * MVP 阶段：使用 Swift 的 `Process` 启动 `lsof -nP -iTCP -sTCP:LISTEN -F`，按行流式读取并解析。
   * 解析输出中的 `p` (PID), `c` (Command), `u` (User), `n` (Name/Address:Port) 字段。
2. **工作目录 (Cwd) 与命令行解析**：
   * 通过 `lsof -p <PID> -a -d cwd -Fn` 获取进程的工作目录。
   * 通过 `ps -p <PID> -o args=` 获取完整的命令行参数。
3. **回收站大小统计**：
   * 遍历 `~/.Trash` 目录，计算所有子文件的大小与文件总数，通过低优先级的 QoS 后台队列完成，防止阻塞 UI 主线程。
4. **能耗与调度设计**：
   * 采用 Grand Central Dispatch (GCD) 对扫描任务使用 `.utility` 优先级的 QoS，由系统智能决定调用能效核 (E-Cores)，在空闲时段使 CPU 负载接近 0%，极其省电。

## 8. 性能与能耗指标

应用应当达到如下性能水平，以符合“轻量、省电”的核心卖点：
* **冷启动时间**：从点击到界面展示不超过 **1 秒**。
* **常驻内存目标**：日常闲置运行时物理内存占用低于 **80MB**。
* **CPU 占用率**：空闲状态下 CPU 占用接近 **0%**；执行流式扫描时瞬间 CPU 不超过 **10%**（且由能效核承载，不唤醒性能核）。
* **UI 流畅度**：列表滚动及滑出抽屉时保持满帧 60fps / 120fps (ProMotion)。

## 9. 验收标准 (Acceptance Criteria)

1. 打开应用，首屏 Dashboard 能够正确显示当前活跃端口数量、对外暴露数量和 `~/.Trash` 大小。
2. 端口列表中，能够正确解析出 `localhost:3000` 并识别出是否是本机专属。
3. 对于在项目目录下启动的 Node.js 进程，能够在列表中识别并显示出其具体的工作目录路径（如 `~/Projects/my-next-app`）和父进程。
4. 点击关闭服务时，发送 `SIGTERM` 信号，且在进程顽固不退出时能够呈现 `Force Stop` 红色按钮进行 `SIGKILL` 强杀。
5. 带有 Apple 官方签名或处于 `/System` 目录下的进程在界面中被保护起来，不可被 Stop。
6. 点击打开 Trash 按钮能够拉起 macOS 的 Finder 并定位到废纸篓。
